Lesson 1: Variables

**Goal:** Learn what variables are and how to store different types of data.

üìù Core Concepts

  • You give a variable a name so you can use its value later.
  • Variables can hold different **data types**, such as numbers, text, or true/false values.

Lesson 2: Functions

üí° Common Mistake: Fixing a Bug

**Task:** This code has a bug! The program crashes because a variable is used before it is created. Fix the code so it runs correctly.

# Find the bug and fix it.
print("Welcome to the adventure, " + character_name + "!")
character_name = "Elara"

Solution:

The bug is a **NameError**. The `print()` statement tries to use the `character_name` variable before it has been assigned a value. Python reads code from top to bottom, so the variable must be created first.

**Fix:** Move the line that creates the variable to be before the `print()` statement that uses it.

character_name = "Elara"
print("Welcome to the adventure, " + character_name + "!")

üìù Core Concepts

  • A **function** is a named block of code that you can run again and again.
  • You **define** a function using the `def` keyword.
  • You **call** a function to run its code by using its name followed by parentheses `()`.
  • To use a function from a different file, you must **import** it first.

üë®‚Äçüè´ Teacher Script

  • **Start with the bug:** "Before we start today, let's look at this broken code. Can you see why it's not working? The computer tries to use `character_name` before we have given it a value. It's like trying to find something in an empty box."
  • **Introduce functions:** "Today we learn a very important idea called **functions**. A function is like a recipe. You write the recipe one time, and you can use it whenever you want without writing all the steps again. This makes your code cleaner and easier to read."
  • **Explain `def`:** "The word `def` is how we write our recipe. We give it a name, like `welcome_traveler()`. The code that belongs to the function is indented, just like the steps in a recipe. We will also add a line that calls our new function. This will tell the computer to run the recipe."
  • **Walk through the building project:** "Open your `my_project.py` file from the last lesson. We will add a new function and a line that calls it. The new lines are highlighted in green. When you run the program, the code in the function will run and print the greeting."

Lesson 4: Numbers

üí° Common Mistake: Fixing a Bug

**Task:** The `attack()` function below needs two arguments, but only one is given. Fix the code so the function is called correctly.

# Find the bug and fix it.
def attack(target_name, damage):
    print("Attacking " + target_name + " for " + str(damage) + " damage.")

attack("Goblin")

Solution:

The bug is a **TypeError**. The function `attack()` is defined to accept two arguments, but the call `attack("Goblin")` only provides one. The number of arguments in the function definition must match the number of arguments in the function call.

**Fix:** Provide a second argument in the function call, such as a number or a variable.

def attack(target_name, damage):
    print("Attacking " + target_name + " for " + str(damage) + " damage.")

attack("Goblin", 10)

üìù Core Concepts

  • **Integers** are whole numbers like `1`, `100`, or `-5`.
  • **Floats** are numbers with a decimal point like `3.14` or `9.99`.
  • You can do math with numbers using operators like `+` (add), `-` (subtract), `*` (multiply), and `/` (divide).

Lesson 5: Boolean Logic - `and`

üí° Common Mistake: Fixing a Bug

**Task:** This code has a bug! It tries to combine a number and a string, which isn't allowed. Fix it so the sentence prints correctly.

Solution:

The bug is a **TypeError**. Python does not know how to join a string (`" is "`) and an integer (`3`) with the `+` operator. To fix this, you must convert the number to a string first.

**Fix:** Use the `str()` function to convert the `pet_age` integer into a string before concatenating it.

pet_name = "Luna"
pet_age = 3
print("My pet " + pet_name + " is " + str(pet_age) + " years old.")

üìù Core Concepts

  • A **boolean** is a data type that can only be either `True` or `False`.
  • The word `and` is a logical operator. When you use `and`, both conditions must be `True` for the final result to be `True`.

Lesson 6: Boolean Logic - `or`

üí° Common Mistake: Fixing a Bug

**Task:** The `is_strong` variable is always `True` because the condition is wrong. Fix it so it correctly checks if the `strength` is greater than or equal to 10.

Solution:

The bug is a **SyntaxError** because the single equal sign `=` is used for assignment, not comparison. To compare if two values are equal, you must use the double equal sign `==`. To check for greater than or equal to, use `>=`.

**Fix:** Change the single equal sign `=` to a greater-than-or-equal-to sign `>=` to correctly compare the values.

strength = 5
is_strong = strength >= 10
print(is_strong)

üìù Core Concepts

  • The word `or` is a logical operator. When you use `or`, only one of the conditions needs to be `True` for the final result to be `True`.

üí° Common Mistake: Fixing a Bug

**Task:** The code below has a bug! It tries to use a `for` loop on a number, which isn't allowed. Fix the code so it correctly counts to 5.

# Find the bug and fix it.
for number in 5:
    print(number)

Solution:

The bug is a **TypeError**. A `for` loop needs a container to loop through, not a single number. To loop a specific number of times, you can use the `range()` function, which creates a list of numbers to loop through.

**Fix:** Use `range(5)` to create a list of numbers from 0 to 4.

for number in range(5):
    print(number)
